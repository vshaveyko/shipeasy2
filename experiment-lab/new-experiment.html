Metrics — what you're measuring
Define what events count as success (goal) and what must not get worse (guardrails).
lightbulb
Pre-register your goal metric before launching. Looking at 20 metrics after the fact and picking the green one is p-hacking — it inflates your false positive rate. Commit to 1 goal, 1–3 guardrails, then launch.

Estimated power
Statistical power = probability of detecting a real effect if it exists. We want ≥80%. Below 80% means you're likely to miss real improvements and incorrectly conclude "no effect." MDE = minimum detectable effect — the smallest improvement you could reliably detect with this sample size. Effects smaller than the MDE will show as gray bars on the dashboard, even if they're real.

Exposure is automatic. Calling getExperiment() is all you need — the SDK logs the exposure event for you. You only need to call track() for the success/metric events.
// ─── Step 1: Get experiment assignment (auto-logs exposure) ─── const exp = await client.getExperiment('exp_your_name') // Render based on group assignment if (exp.group === 'test') { renderNewVersion(exp.params) } else { renderControl() } // ─── Step 2: Track success events ───────────────────────────── // Call client.track() when the success event happens client.track('purchase_completed', { value: order.total }) // Guardrail: track page load time client.track('page_loaded', { value: performance.now() })
The analysis runs nightly at 04:00 UTC. Results appear on the dashboard the next morning.
