The 5‑minute path

Three commands. Three files. One coffee. By the end of this page your app is shipping in 12 languages with translation memory, a glossary, and an inline editor for non‑developers.

Quickstart — from zero to localized

~5 minutes · no card required
01 · INSTALL

Add Polylang to your repo

One CLI flag detects your framework, source files, and existing locale folders.

$npx polylang init ⎘
02 · TRANSLATE

Pick languages, hit go

Claude streams translations file‑by‑file. Glossary, tone and brand voice respected.

$polylang translate --to es,fr,de,ja ⎘
03 · SHIP

Wire it up & deploy

Drop in the <Polylang/> provider. Strings switch on locale.

$polylang sync && vercel deploy ⎘

Install

Polylang is a single CLI plus a tiny runtime. Pick the package manager you already use — everything else is detected automatically.

npm
pnpm
yarn
bun
$ npm install --save-dev polylang && npx polylang init Copy

The init script writes one config file (polylang.config.ts) and creates a locales/ folder seeded with your default language. Nothing else.

BYOK or hosted, your call. Polylang works with your own Anthropic API key (zero markup, zero data retention) or our hosted plan with a generous free tier. Switch any time with polylang.config.ts.

Configure your project

The config file is plain TypeScript — no YAML, no JSON gymnastics. Most teams ship the defaults and never touch it again.

📄 polylang.config.ts typescript ⎘ Copy 
 1import { defineConfig } from "polylang";
 2
 3export default defineConfig({
 4  source: "en",
 5 6  targets: ["es", "fr", "de", "ja", "pt-BR"],
 7  files:   ["app/**/*.{ts,tsx}", "content/**/*.md"],
 8
 9  // Brand voice — Claude reads this on every translation.
10  tone: "warm, technical, never marketing-speak",
11  glossary: {
12    "Polylang": "Polylang",   // never translate
13    "ship":     "deploy",     // disambiguate
14  },
15
16  provider: { kind: "anthropic", model: "claude-haiku-4-5" },
17});

What just happened?

The highlighted line tells Polylang which languages to ship. Everything else is optional — but the tone and glossary fields are what separate Polylang output from machine translation: Claude actually reads them before translating each string.

Core concepts

Four ideas worth ten minutes — they make the rest of the docs obvious.

Translation memory →

Edit a string once, reuse it everywhere. Claude is asked only for net‑new content, so re‑translation is fast and cheap.

~94% hit rate·cached at the edge

Inline edit →

Hold ⌥ in any preview and click any string to rewrite it across every language. Ships as a dev‑only devtool.

per-target overrides·versioned

API · defineConfig

The full type lives in polylang/types. These are the fields you'll actually touch.

Field
Type
Description
sourcerequired
Locale
The language your source code is written in. Usually "en".
targetsrequired
Locale[]
BCP‑47 codes to translate into. Use region tags (pt-BR) when register matters.
files
string[] ?
Globs to scan for user‑facing strings. Defaults to your framework's source dirs.
tone
string ?
One‑line voice guide. Claude reads this every translation. Keep it short — tonal, not exhaustive.
glossary
Record<string,string> ?
Terms to lock or remap. Map a term to itself to mark do‑not‑translate.
provider
Provider ?
Defaults to hosted. Swap to { kind: "anthropic", apiKey: env.ANTHROPIC_KEY } for BYOK.

Use it in your app

Polylang ships a thin runtime — about 1.4 kB gzipped. The useT hook reads the locale from your router and returns translations from the bundle generated at build time.

📄 app/(marketing)/page.tsx tsx ⎘ Copy 
 1import { useT } from "polylang/react";
 2
 3export default function Page() {
 4  const t = useT();
 5  return (
 6    <section>
 7      <h1>{t("Localize, as easy as asking Claude.")}</h1>
 8      <p>{t("Ship in 12 languages without ever leaving your editor.")}</p>
 9    </section>
10  );
11}

Notice — there is no en.json file you wrote by hand, no string keys, no i18n.t("hero.title"). The English is the source of truth and the key. Polylang generates a stable hash and keeps every locale's bundle in sync.

Don't translate user‑generated content this way. useT is for static UI strings extracted at build time. For UGC (chat messages, reviews, support tickets), use the runtime /translate endpoint with our streaming SDK.
