FAQ

You probably want to know.

How does the MCP install actually work?

One command — claude mcp add shipeasy — registers four tools with Claude. We use your project context to wire experiments into the right files. No YAML, no dashboard configuration.

Is Shipeasy hosted, or can I self-host?

Both. Hosted is the default. Self-hosted is a one-binary deployment on the Enterprise plan, with full parity and an air-gapped mode.

Which stats engine do you use?

Sequential testing by default — you can peek whenever, without inflating false-positive rates. Bayesian and frequentist are available per-experiment.

What if Claude makes a mistake?

Every action is reversible and logged. Killswitches and experiment changes go through a PR or a tool call you can undo. Nothing touches production unreviewed.

Which frameworks do you support?

JS/TS, Python, Go, Ruby, Rust, and Swift. The SDK is a thin client; the heavy lifting happens server-side.
