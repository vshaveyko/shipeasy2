## Four primitives, one mental model

<TileGrid>
  <Tile
    href="/flags-experiments/gates"
    icon="⌗"
    title="Gates — boolean flags"
    meta={<><span><b>~1ms</b> evaluation</span><span>·</span><span>no per-request fetch</span></>}
  >
    Targeted rollouts: enable for `country=US`, ramp `5% → 100%`. The default tool when you want to
    ship code without showing it to everyone.
  </Tile>
  <Tile
    href="/flags-experiments/configs"
    icon="⚙"
    title="Configs — typed values"
    meta={<><span><b>JSON</b> schema-validated</span><span>·</span><span>typed access</span></>}
  >
    String, number, boolean, JSON values you change without a redeploy. Use when the question is
    *"what value?"* not *"on or off?"*.
  </Tile>
  <Tile
    href="/flags-experiments/killswitches"
    icon="⏻"
    title="Killswitches — break-glass"
    meta={<><span><b>1-click</b> disable</span><span>·</span><span>auditable</span></>}
  >
    A flag with a single job: disable a system in an incident. Separate from gates so on-call
    doesn't have to think about rollout percentages at 3am.
  </Tile>
  <Tile
    href="/flags-experiments/experiments"
    icon="◈"
    title="Experiments — A/B tests"
    meta={<><span><b>p-values</b> + 95% CIs</span><span>·</span><span>daily updates</span></>}
  >
    Universes, holdouts, multivariate variants. Automated p-values and 95% confidence intervals
    delivered to the dashboard each day.
  </Tile>
</TileGrid>

<Callout type="info" title="Not sure which to use?">
  See the [decision tree](/flags-experiments/decision) — one diagram that maps "I want to do X" to
  the right primitive.
</Callout>

## How it works

Every primitive is read by your SDK with sub-millisecond latency — no round-trip on the hot path.
Writes you make in the dashboard or CLI propagate worldwide in under a second.
