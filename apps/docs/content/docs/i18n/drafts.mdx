---
title: Drafts
description: Translation proposals — manual or AI — that go through review before publishing.
---

Translations don&apos;t go straight into a profile. They land first as a **draft**: a set of proposed values for one profile, attributable to a source (you, an AI run, an imported file) and reviewable inline.

## Lifecycle

```
created → (review) → merged → (publish)
                  └→ abandoned
```

| State       | What it means                                             |
| ----------- | --------------------------------------------------------- |
| `open`      | Draft exists, nothing merged yet.                         |
| `merged`    | Values were promoted into the profile. Still unpublished. |
| `published` | The profile (with these values) was published to KV.      |
| `abandoned` | Discarded without merging. Auto-deleted after 30 days.    |

A profile can have one open draft at a time per source. Multiple sources are fine — you can have an AI draft and a manual draft running in parallel and pick keys from each.

## Creating a draft

### From AI

```bash
shipeasy i18n translate --profile fr:prod --ai
```

The CLI calls Anthropic Claude (using your project&apos;s API key — see [Translations](/i18n/translations) for setup) and creates a draft with proposed values for every key missing from the target profile. Existing values are not overwritten unless you pass `--reflow`.

### From a JSON file

If you already have translations (a contractor delivered a file, you exported from another tool):

```bash
shipeasy i18n translate --profile fr:prod --import ./french-translations.json
```

The file is a flat `{ "key": "value" }` map.

### Manual entry

In the dashboard, **Drafts → New** opens an empty draft for the chosen profile. You can paste keys and values, or use the inline editor.

## Reviewing a draft

Open **String Manager → Drafts → fr:prod**. You see a three-column view:

| Source (en:prod)  | Proposed (fr:prod)  | Decision               |
| ----------------- | ------------------- | ---------------------- |
| Welcome back      | Bienvenue à nouveau | ✓ Approve              |
| Pay               | Régler              | ✗ Reject (edit inline) |
| 3 unread messages | 3 messages non lus  | ✓ Approve              |

You can:

- **Approve** a row — the value moves to the merged set.
- **Edit inline** — fix the AI&apos;s output before approving.
- **Reject** — keep the existing profile value (or the source fallback).
- **Mark for translator** — flag a row that needs human attention before merge.

The dashboard shows the source value, any description on the key, and the AI&apos;s confidence (when sourced from AI).

## Merging

When you&apos;re satisfied, click **Merge draft**. Approved values are promoted into the profile. The draft moves to `merged`.

**Merging does not publish.** This is deliberate — merging is "I&apos;ve reviewed; the profile is internally correct". Publishing is "this version is going to users". Decoupling them lets you stage a complete merge (often dozens of keys at once) and publish atomically:

```bash
shipeasy i18n publish --profile fr:prod
```

You can also auto-publish on merge in **Project → String Manager → Settings** if you trust the review step entirely.

## Side-by-side review

For target locales you don&apos;t speak, the dashboard offers a **back-translation** view. The AI translates the proposed value back to the source locale; you compare the back-translation against the original. Mismatches usually mean the proposed translation drifted.

```
Original:        "Pay"
Proposed (FR):   "Régler"
Back-translation: "To settle / pay (a bill)"  → ✓ acceptable
```

## Conflict resolution

If two open drafts touch the same key, the latest merge wins; the other draft&apos;s row for that key is marked **conflicted** and you re-decide on next review.

## CI integration

For teams running translation contracts, the `--ci` mode generates a draft, prints a summary, and exits non-zero if the draft is empty (nothing to translate) — handy in scheduled cron jobs that translate any new keys nightly.

```bash
shipeasy i18n translate --profile fr:prod --ai --ci
```

## Abandoning

```bash
shipeasy i18n drafts abandon <draft-id>
```

Or from the dashboard. Abandoning frees the profile for a new draft. The abandoned values remain in the dashboard&apos;s history for 30 days in case you change your mind.
