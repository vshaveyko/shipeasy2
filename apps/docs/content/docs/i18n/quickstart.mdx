This walks through the full onboarding for a fresh frontend project. We&apos;ll discover translatable strings, push them to a source profile, generate French translations with AI, and publish.

If you&apos;d rather have an AI agent do this for you, see the [LLM Guide](/llms/i18n-strings) — it&apos;s the same flow, written for Claude Code / Cursor / Windsurf.

## Prerequisites

- A frontend project with English copy you want to translate. Next.js, Vite, Remix, etc.
- The CLI installed and logged in: `npm install -g @shipeasy/cli && shipeasy login`.

<Steps>
  <Step title="Install the loader script">
    The loader is a tiny `<script>` that fetches your published label bundle at runtime. It has zero JS dependencies and benefits from infinite CDN caching.

    ```bash
    shipeasy i18n install-loader --profile en:prod
    ```

    The CLI auto-detects your framework and patches the right file:

    | Framework | Patches |
    | --- | --- |
    | Next.js App Router | `src/app/layout.tsx` (inside `<head>`) |
    | Next.js Pages | `pages/_document.tsx` |
    | Vite + React | `index.html` |
    | Remix | `app/root.tsx` |
    | Anything else | Prints the tag, you paste it |

    The command also creates a **client SDK key** scoped to the project, hard-coded into the script tag. The key only allows reads of published label bundles, nothing else.

  </Step>

  <Step title="Create the source profile">
    A profile is a `locale:environment` pair. Create the one matching the language already in your code:

    ```bash
    shipeasy i18n profiles create --name en:prod --locale en --environment prod
    ```

    For non-English source, use the right BCP-47 code (`fr`, `de`, `ja`, …).

  </Step>

  <Step title="Scan & codemod">
    Discover strings in your source files. For JSX/TSX we can rewrite them automatically — _codemod_:

    ```bash
    shipeasy i18n scan ./src --rewrite --profile en:prod
    ```

    The codemod replaces:

    ```tsx
    <h1>Welcome back</h1>
    ```

    with:

    ```tsx
    <h1>{t("welcome_back", "Welcome back")}</h1>
    ```

    and adds the import. For non-JSX languages (Vue, Svelte, plain HTML, server templates), the scan reports candidates and you wrap them by hand.

    A `i18n-codemod-review.json` report is written at the project root listing strings the codemod skipped (dynamic interpolations, rich HTML, etc.).

  </Step>

  <Step title="Push keys">
    ```bash
    shipeasy i18n push --profile en:prod --source codemod
    ```

    Pushes every discovered key to the dashboard. Verify in **String Manager → Label keys**.

  </Step>

  <Step title="Add target locales">
    Say you want French and German:

    ```bash
    shipeasy i18n profiles create --name fr:prod --locale fr --environment prod
    shipeasy i18n profiles create --name de:prod --locale de --environment prod
    ```

  </Step>

  <Step title="Generate AI translations">
    ```bash
    shipeasy i18n translate --profile fr:prod --ai
    shipeasy i18n translate --profile de:prod --ai
    ```

    Each command creates a **draft** in the dashboard with proposed translations for every key missing from the target profile. The AI uses the source value and any descriptions you&apos;ve added on each key.

  </Step>

  <Step title="Review">
    Open **String Manager → Drafts → fr:prod**. You see every key, the source value, and the proposed translation, side by side. Edit any value inline. When you&apos;re satisfied, click **Merge draft**.

    Merging promotes the values to the profile. It does **not** publish — that&apos;s a separate, deliberate step.

  </Step>

  <Step title="Publish">
    ```bash
    shipeasy i18n publish --profile fr:prod
    shipeasy i18n publish --profile de:prod
    ```

    Each publish writes the chunked label bundle to KV and purges the relevant CDN URLs. New copy is live worldwide in seconds.

  </Step>

  <Step title="Switch locale at runtime">
    The loader reads the active locale from your URL or a cookie. Most frameworks already provide this; the SDK exposes a helper:

    ```ts
    import { i18n } from "@shipeasy/sdk/client";
    i18n.setLocale("fr");
    ```

    The loader fetches the new bundle, swaps the strings, and re-renders.

  </Step>
</Steps>

## Validate before deploy

In CI, fail the build if any code reference is missing from the source profile:

```bash
shipeasy i18n validate --profile en:prod ./src
```

Add this to your pre-merge checks and you&apos;ll never ship an "untranslated string" bug.
