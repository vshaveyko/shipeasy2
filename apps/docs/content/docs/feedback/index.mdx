## Two record types

<TileGrid>
  <Tile href="#bugs" icon="◎" title="Bugs — what's broken">
    Title, page URL, optional stack trace and screenshot. Status flow: `open` → `triaged` →
    `resolved`.
  </Tile>
  <Tile href="#feature-requests" icon="✦" title="Feature requests — what's missing">
    Title, importance (`nice-to-have` / `important` / `critical`), votes from other users. Status
    flow: `open` → `planned` → `shipped`.
  </Tile>
</TileGrid>

## How it works [#how-it-works]

A user hits **Report bug** in your app. The SDK sends the report to ShipEasy. You see it in the
dashboard within seconds. Optional webhooks forward it to Slack / Linear / Jira so you don't have
to live in a second tool.
