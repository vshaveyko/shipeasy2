ShipEasy is built so that an AI coding agent (Claude Code, Cursor, Windsurf, OpenCode, …) can do most of the boring setup work for you. Hand it a guide URL and it executes a complete onboarding workflow — installing the SDK, scanning your code, wiring the loader, pushing keys, publishing.

## How to use these guides

<Steps>
  <Step title="Install the MCP server">
    The agent uses ShipEasy&apos;s [MCP server](/get-started/mcp) to call typed tools. One command sets it up:

    ```bash
    shipeasy mcp install
    ```

    Or wire it by hand — see the MCP page.

  </Step>
  <Step title="Log in once">
    ```bash
    shipeasy login
    ```

    The MCP tools pick up your credentials automatically. The agent never sees your token.

  </Step>
  <Step title="Hand the agent a URL">
    Open your AI assistant and say:

    > "Follow https://docs.shipeasy.ai/llms/i18n-strings to onboard this project to ShipEasy String Manager."

    The agent reads the guide, calls the right MCP tools in the right order, asks you only for the things that need a human (e.g. clicking a browser link to authenticate).

  </Step>
</Steps>

## Why URLs, not packaged prompts

The guides are plain MDX. Anyone — human or agent — can read them. They evolve with the platform; the URL is stable, the content is current. If you fork a guide for your team&apos;s conventions, you&apos;re welcome to host your own — the MCP tools and shapes are public.
