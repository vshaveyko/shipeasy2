## The 5-minute path

<Quickstart title="From zero to ShipEasy" time="~5 minutes · no card required">
  <QuickstartStep
    num={1}
    label="INSTALL"
    title="Add the SDK and CLI"
    cmd="npm i @shipeasy/sdk && npm i -g @shipeasy/cli"
  >
    One package for server + browser. The CLI logs in via your browser — no env tokens to wrangle.
  </QuickstartStep>
  <QuickstartStep
    num={2}
    label="WIRE"
    title="Configure once, use everywhere"
    cmd="shipeasy login && shipeasy bind"
  >
    Flags, experiments, translations, and feedback share a single API key. One <code>shipeasy()</code>
    call boots all of them.
  </QuickstartStep>
  <QuickstartStep
    num={3}
    label="SHIP"
    title="Flip a flag, ship a translation"
    cmd="shipeasy flags create checkout-v2 --rollout 5"
  >
    Changes you make in the dashboard go live worldwide in under a second.
  </QuickstartStep>
</Quickstart>

## Why ShipEasy

- **One SDK** for gates, configs, experiments, translations, and feedback. Server _and_ client.
- **One CLI** (`shipeasy`) for everything you can do in the dashboard.
- **One MCP server** so your AI agent can do the boring setup for you.
- **Sub-millisecond evaluation**: flags resolve in your code without a network round-trip.
- **Export your data**: results and events are yours, not locked into our dashboard.
