## The 5-minute path

<Quickstart title="From zero to ShipEasy" time="~5 minutes · no card required">
  <QuickstartStep
    num={1}
    label="INSTALL"
    title="Add the SDK and CLI"
    cmd="npm i @shipeasy/sdk && npm i -g @shipeasy/cli"
  >
    One package for server + browser. The CLI logs in via your browser — no env tokens to wrangle.
  </QuickstartStep>
  <QuickstartStep
    num={2}
    label="WIRE"
    title="Configure once, use everywhere"
    cmd="shipeasy login && shipeasy init"
  >
    Flags, experiments and i18n share a single API key. One <code>shipeasy()</code> call boots all
    three.
  </QuickstartStep>
  <QuickstartStep
    num={3}
    label="SHIP"
    title="Flip a flag, ship a translation"
    cmd="shipeasy gate create checkout-v2 --rollout 5"
  >
    Reads happen from KV at the edge. Writes purge surgically — visible worldwide in under a second.
  </QuickstartStep>
</Quickstart>

## Why ShipEasy

<Callout type="success" title="Built for the edge">
  Reads happen at Cloudflare&apos;s edge from KV with infinite CDN TTL — sub-millisecond, worldwide.
  Writes are explicit, fast, and surgical: when you flip a flag, ShipEasy purges the affected blob
  and the next request anywhere on Earth sees the new value within a second.
</Callout>

- **One SDK** for gates, configs, experiments, and translations. Server _and_ client.
- **One CLI** (`shipeasy`) for everything you can do in the dashboard.
- **One MCP server** so your AI agent can do the boring setup for you.
- **Stateless evaluation**: flags evaluate locally on the server, no per-request round-trip.
- **No vendor lock-in for stats**: experiment results are written back to your project as plain rows you can export.
